# Phase 1 Lesson 1: 向量、Embedding、相似度、ANN 基础

## 1. 先建立一个正确直觉

### 向量是什么
向量在这里可以先理解成一个定长数字数组，例如：

```text
[0.12, -0.44, 1.31, ...]
```

这串数字不是给人直接看的，而是给模型和检索系统用来表示“对象特征”或“语义位置”的。

对象可以是：
- 一段文本
- 一张图片
- 一首音频
- 一个商品
- 一个用户
- 一条日志

### Embedding 是什么
Embedding 是一个映射过程：

```text
原始对象 -> 模型编码 -> 向量
```

例如：
- “今天天气很好” -> 一个文本向量
- “晴天白云的照片” -> 一个图片向量

Embedding 模型做的事，本质上是把原始对象投影到一个高维空间里，让“语义相近”的对象尽量靠近。

### 为什么向量能支持语义检索
因为 embedding 模型训练时，会尽量让：
- 意思接近的对象，向量更近
- 意思不相关的对象，向量更远

所以检索时你不再只看关键词是否相同，而是看“向量空间里的距离是否接近”。

## 2. 三个最核心概念

### 概念 A: 距离或相似度
向量检索本质上是在问：

```text
谁和查询向量最像？
```

常见度量：
- Cosine similarity: 看方向是否接近
- L2 distance: 看欧氏距离是否接近
- Inner product: 看点积是否大

### 概念 B: TopK
通常不会只找一个最相近的向量，而是找前 K 个最近邻：

```text
query -> top 5 nearest vectors
```

### 概念 C: ANN
如果数据量很大，逐条和所有向量精确比较会非常慢。

所以实际系统常用 ANN:
- Approximate Nearest Neighbor
- 近似最近邻检索

它的核心思想是：
- 不保证百分之百找出绝对最优答案
- 但显著降低延迟和成本
- 在大规模场景下更实用

## 3. Cosine、L2、Inner Product 怎么理解

### Cosine similarity
看两个向量夹角是否接近。

适合场景：
- 更关心语义方向
- 不太关心向量长度
- 文本 embedding 场景里很常见

直觉：
- 两个人说话内容很像，即使篇幅长短不同，方向也可能接近

### L2 distance
看两点在空间中的直线距离。

适合场景：
- 向量值的绝对位置本身有意义
- 模型或索引流程按欧氏空间假设设计

直觉：
- 地图上两个点离得越近，L2 越小

### Inner Product
看点积大小。

适合场景：
- 需要结合向量长度与方向
- 某些推荐系统或 MIPS 类检索问题

要点：
- 如果向量先做归一化，inner product 和 cosine 在很多场景里会非常接近

## 4. Dense / Sparse / Multi-vector

### Dense vector
大多数维度都有值。

例子：
```text
[0.21, -0.93, 0.48, 1.02]
```

常见于：
- 神经网络 embedding
- 文本、图像、音频的语义表示

### Sparse vector
只有少数维度非零。

例子：
```text
{12: 0.8, 509: 0.3, 2048: 1.1}
```

常见于：
- BM25
- TF-IDF
- 稀疏编码特征

特点：
- 更容易保留“关键词强信号”
- 往往适合和 dense 检索混合使用

### Multi-vector
同一个对象，不只存一个向量，而是存多个向量。

常见于：
- 一个文档拆成多个 chunk
- ColBERT 这类 token-level late interaction 方法
- 多字段分别编码，例如标题、正文、标签

意义：
- 一个对象内部可能有多个语义视角
- 单个向量有时表达不够细

## 5. 一个极简例子

假设库里有三条文本：
- A: “如何安装 Python”
- B: “Python 环境配置教程”
- C: “今天午饭吃什么”

查询是：
- Q: “怎么配置 Python 开发环境”

关键词检索未必稳定，因为 Q 和 A/B 的字面词不完全一致。
但如果 embedding 模型足够好，Q 的向量会更接近 A 和 B，而远离 C。

这就是语义检索。

## 6. 你现在必须避免的 5 个误区

1. 向量不是“原始数据”，而是模型编码后的表示。
2. 向量数据库不是“大模型”，它只是负责高效存储、索引、检索这些向量。
3. 向量检索不是天然比关键词检索更好，它们常常应该混合。
4. Embedding 质量通常比索引参数更影响最终效果。
5. ANN 不是错误结果，而是在召回率、延迟、成本之间做工程权衡。

## 8. 快速自测

### 题 1
“向量数据库”的核心职责是什么？

参考答案：
高效存储向量、建立索引、执行近邻检索，并和标量过滤、排序、元数据结合返回结果。

### 题 2
“Embedding 模型”的核心职责是什么？

参考答案：
把文本、图片等原始对象编码为可用于相似度计算的向量表示。

### 题 3
什么时候应该优先怀疑 embedding，而不是优先调 ANN 参数？

参考答案：
当语义相近的内容总体都召回不上来时，往往先怀疑表示质量，而不只是索引参数。
